LLDP 拓扑工具使用手册 v3

概览

一句话：`lldp.html` 是完整运维页，负责把 SQL / CLI / NDMP / CSV / Json 多来源拓扑数据统一成一张可操作的图，并在同一页面完成利用率分析、Zabbix 告警、设备详情、路径查询、手工补图和多格式导出。

  • 采集 SQL、CLI、NDMP、CSV、Json 五类导入入口统一收口。
  • 整理 支持节点组、分层、分区、路径抽取、手工节点、Cloud 节点和手工链路。
  • 分析 支持链路利用率、拥塞分析、负载分担分析、告警高亮和 Netlog 跳转。
  • 落地 支持 PNG、draw.io、Mermaid、链路汇总 CSV、设备详情 CSV、Json 会话和服务器状态快照。

页面分工

  • `lldp.html` 采集 + 分析 + 运维。适合实时导入、链路分析、告警查看、设备详情查询和导出结果。
  • `index.html` 编辑 + 整理 + 展示。适合在已有 CSV / Json 基础上做版式整理、路径抽取、汇报图加工。
  • 使用建议 需要实时采集或联动外部系统时用 `lldp.html`；只想整理已有数据时用 `index.html`。

启动方式

  • 推荐入口 通过服务访问：http://127.0.0.1:18080/lldp.html。
  • 原因 SQL、CLI、NDMP、状态快照、临时 CSV、设备详情、Zabbix 等能力都依赖后端服务。
  • 脚本能力 当前启动脚本支持 start、stop、status、restart、logs。
  • 依赖处理 执行 start 时会自动检查并补装缺失的 fastapi、uvicorn、pymysql。
  • 配置文件 SQL / CLI 默认读取 .env.mysql，可通过 DB_ENV_FILE 改指其他配置文件。

本地启动（推荐）

cd lldp_topology
python3 -m venv ../.venv
source ../.venv/bin/activate
pip install -r requirements.txt
cp .env.mysql.example .env.mysql
./start_lldp_service.sh start
  • 状态 ./start_lldp_service.sh status
  • 日志 ./start_lldp_service.sh logs
  • 重启 ./start_lldp_service.sh restart
  • 停止 ./start_lldp_service.sh stop
  • 访问 启动后打开 http://127.0.0.1:18080/lldp.html

Docker 启动

cd lldp_topology
cp .env.mysql.example .env.mysql
docker compose up -d --build
  • 日志 docker compose logs -f
  • 停止 docker compose down
  • 环境变量 docker-compose.yml 可传入数据库、Zabbix、CLI 跳板相关配置。
  • 持久化 tmp_csv/ 和 state_snapshots/ 挂载为卷，便于保留状态和临时结果。

导入模式

SQL 导入

  • 入口 选择 SQL 导入，点击「导入」或「追加」。
  • 输入 使用 start_hostname 作为起始设备名称。
  • 执行方式 采用异步任务模式，前端轮询任务状态，后端输出临时 CSV。
  • 取消 支持「取消查询」，避免长查询只能重启服务。
  • 结果 「导入」覆盖当前工作集；「追加」把新结果并入当前图。

CLI 导入

  • 入口 选择 CLI 导入，点击「导入」或「追加」。
  • 关键参数 起始设备地址、递归次数、并发数、SSH 用户名 / 密码、跳板地址、SSH 命令、登录超时、命令超时。
  • 递归限制 可勾选「只递归查询 172.* 网段」，避免把 10.* / 143.* 等地址继续扩散。
  • Ping 预检 可选「先 Ping 预检（2 个包，失败即跳过）」以减少不可达设备等待。
  • 任务模式 CLI 采用异步任务执行，页面会显示累计完成、本层设备数、并发、当前设备和已耗时。
  • 取消 支持「取消查询」，并在取消时尽量保留已获取的部分结果。
  • 追加策略 右键追加在 CLI 模式下固定只跑 1 轮，避免长时间递归。

NDMP 导入

  • 入口 选择 NDMP 导入，输入设备名称或 IP。
  • 查询字段 支持自动识别、强制按 device_hostname、或强制按 device_ip。
  • 轮循次数 支持 1 到 3 层。
  • 状态过滤 默认仅查 link_status = up，也可以取消勾选以查询非 up 历史状态。
  • 用途 适合快速从接口与邻接字段生成拓扑，而不依赖 CLI 跳板登录。

CSV / Json 导入

  • CSV 支持文件导入、追加、以及直接粘贴 CSV 内容。
  • Json 用于恢复完整前端会话，包括数据、坐标、节点组、颜色、描述和部分视图状态。
  • 追加 新数据尽量保留旧节点坐标，新节点按当前图继续并入。

拓扑生成与整理

  • 生成拓扑 统一把当前工作集转成节点和链路，并处理设备归一、链路去重、多链路合并和坐标继承。
  • 局部操作 支持「重排选中设备」，只调整选中节点，不重建整图。
  • 布局与对齐 支持上下分层、分区、水平对齐、垂直对齐、左右上下对齐、水平 / 垂直等间距。
  • 路径查询 支持源节点、目的节点、必经点、备份路径，并可只保留路径节点。
  • 节点组 支持多选合组、组视图、组名修改、移出节点组、返回主视图。
  • 手工建模 支持添加普通节点、Cloud 节点、手工链路和链路描述。
  • 多链路控制 可在聚合显示和逐条物理链路显示之间切换。

视图过滤与交互

  • 定位设备 输入表达式后快速查找并选中设备。
  • 不显示 / 只显示 使用统一表达式语法：逗号分层、斜杠 OR、& 全命中。
  • 只显示 172 设备 仅对普通节点生效，节点组不参与该判断。
  • 单链路节点 自动选中当前可见拓扑中只连到 1 台邻居设备的节点，并可联动标记节点组。
  • 删除与恢复 支持删除节点、仅保留选中、仅保留邻接节点、恢复删除设备。
  • 撤销 / 恢复 支持有限步数的视图级回退，用于节点移动、删除、对齐、颜色、手工补图等操作。
  • 固定位置 右键可固定或取消固定节点，避免整理后再次被局部操作带动。

链路 / 告警 / 设备详情

链路利用率

  • 查询入口 支持工具区查询，也支持右键「当前利用率」和「区间利用率」。
  • 数据源 优先走 Zabbix；若未命中且已配置 CLI 账号密码，可回退到 CLI。
  • 查询口径 支持 TX、RX、双向较小值 MIN(TX,RX)、双向较大值 MAX(TX,RX)。
  • 展示控制 支持链路拥塞分析、负载分担分析、显示利用率标签、显示 TX/RX、显示总带宽、显示链路描述。
  • 利用率过滤 可按阈值只保留高于阈值的链路和相关节点。

Zabbix 告警

  • 查询模式 支持当前未恢复告警和历史告警。
  • 确认状态 支持全部、仅未确认、仅已确认。
  • 活动告警显示 勾选后，未确认活动告警设备会闪动；仅已确认告警设备不闪动，但名称文字变红并显示告警数量；无告警设备保持正常颜色。
  • 时间过滤 「告警时间过滤」只过滤当前图上的活动告警显示，不会重新查询 Zabbix。
  • 右键 支持「查看告警」和「刷新告警」，刷新只针对当前选中设备。
  • 导出 告警结果支持 CSV 导出。

Netlog 跳转

  • 入口 有已缓存告警的设备右键可见「查询 Log」。
  • 行为 不直接执行日志任务，而是打开 Netlog 创建页，并预填主机名、告警时间窗口和 SQL only 参数。
  • 时间窗 单设备按最近告警时间上下 15 分钟；多设备按各自最近告警的整体包络计算时间窗。
  • 前提 要求 Netlog 服务可访问，并且当前设备已经有已缓存告警结果。

设备详情

  • 查询 支持对选中设备或当前可见设备批量查询详情。
  • 复制 右键「复制详情」可将已缓存详情直接复制出来，适合贴到工单或文档。
  • 导出 「导出设备详情」支持字段复选，并在导出前自动补查缺失设备。

导出与状态

  • 导出 PNG 适合截图、汇报和即时分享。
  • 导出链路汇总 会按当前显示状态导出设备级或端口级链路汇总。
  • 导出 draw.io 便于后续在 draw.io 继续排版。
  • 导出 Mermaid 当前页面导出为 .mmd 文件。
  • 导出 Json 用于恢复当前前端工作现场。
  • 保存状态 / 导入状态 服务端保留最近 15 个状态，适合阶段性保存和回滚。
  • 清理 「删除临时数据」会清理临时 CSV、状态快照和服务日志；「删除连接配置」只删除浏览器本地保存的 SQL / CLI / Zabbix 设置。

推荐工作流

  • 启动服务并打开 lldp.html。
  • 先用 SQL / CLI / NDMP / CSV / Json 任一方式导入基线拓扑。
  • 点击「生成拓扑」，再通过分层、分区、只显示 / 不显示、路径查询收口重点区域。
  • 需要补图时，用节点组、Cloud 节点、手工链路和链路描述补充语义。
  • 需要分析时，查询链路利用率、Zabbix 告警、设备详情，必要时从告警设备跳转到 Netlog。
  • 阶段性成果可保存状态，最终导出 PNG / draw.io / Mermaid / Json / 链路汇总 / 设备详情。
