LLDP 拓扑工具使用手册 v2

概览

一句话：支持 CSV / SQL / CLI 三种导入，自动去重生成拓扑，支持分层布局、过滤定位、多选操作、链路利用率刷新、导出 PNG 与会话快照。

启动方式

  • 仅离线导入 CSV 直接打开 lldp.html。
  • 启用 SQL/CLI 需启动服务后访问 http://127.0.0.1:18080/lldp.html。
  • 依赖安装 pip install fastapi uvicorn pymysql。
  • 数据库配置 复制 .env.mysql.example 为 .env.mysql 并填写参数。

一键启动（推荐）

  • 脚本 /Users/zhangwei/python/lldp_topology/start_lldp_service.sh
  • 启动 cd /Users/zhangwei/python/lldp_topology && ./start_lldp_service.sh start
  • 状态 ./start_lldp_service.sh status
  • 日志 ./start_lldp_service.sh logs
  • 重启 ./start_lldp_service.sh restart
  • 停止 ./start_lldp_service.sh stop

SQL 导入（start_hostname）

  • 入口 导入方式选 SQL 导入，可点「导入输入」或「追加输入」。
  • 输入 设备名 start_hostname。
  • 执行方式 前端提交异步任务并轮询状态，任务完成后按下载地址读取结果，避免大结果直接回传导致超时。
  • 行为 「导入输入」会覆盖当前结果；「追加输入」会把本次结果合并到当前图中补链路。
  • 下载 支持下载本次查询的临时 CSV。
  • 临时文件 写入 tmp_csv/。

CLI 导入（SMC 跳板）

  • 入口 导入方式选 CLI 导入，可点「导入输入」或「追加输入」。
  • 参数 设备地址、递归层数、并发数、SSH 账号、SMC 参数、超时参数。
  • 本地记忆 自动记住上次输入（密码除外）。
  • 会话密码复用 页面不刷新时，CLI 可复用本次会话里最近一次输入的密码。
  • 流程 跳板登录 → 设备登录 → 厂商识别 → LLDP 采集 → 递归。
  • 过滤递归 支持「仅递归 172.* 网段」，跳过 10.* / 143.*。
  • 调试 支持下载 CLI 调试日志。

数据导入

  • CSV 选择文件、拖拽、或粘贴内容。
  • 去重 按 localhostname/localinterface/remotehostname/remoteinterface 去重。
  • 追加 追加 CSV 时保留当前坐标。
  • 去后缀 可去掉 hostname 域名后缀。

布局与分层

  • 布局 上下分层 / 强分散 / 圆形。
  • 分层关键字 逗号分层，斜杠同层，& 同层内全命中。示例：EDGE&CORE,CORE,AGGR/SPINE,LEAF/tor
  • 默认 布局模式默认上下分层。

显示与交互

  • 标签 节点名全名/缩略，端口标签显示/隐藏。
  • 过滤 「不显示」关键词回车生效。
  • 定位 「定位设备」关键词回车多选。
  • 右键 固定/删除/复制名称/复制地址。
  • 右键追加查询 按当前导入方式执行一次追加查询；SQL 用设备名，CLI 优先用管理地址。
  • 单轮追加 右键追加查询固定只跑 1 轮，避免长时间递归。
  • 恢复 支持恢复删除节点。

链路显示

  • 多链路合并 同一对节点可合并显示并统计条数。
  • CLI 链路利用率 采用任务化执行（提交任务 + 轮询进度），可查看累计耗时并下载调试日志。
  • 缓存策略 已成功链路结果写入缓存，仅新增链路增量查询。

会话与导出

  • 图片 导出 PNG。
  • 会话 导出会话 / 导入会话（包含当前数据与坐标，可继续后续追加查询与操作）。

常用操作流

  • 启动服务 → 打开页面 → 选择 SQL/CLI/CSV 导入。
  • 先用「导入输入」建立基线拓扑，再用「追加输入」补充新链路。
  • 需要针对某个设备补链路时，右键节点点「追加查询」（单轮）。
  • 生成拓扑后，用分层关键字和不显示规则调整视图。
  • 需要链路占用率时执行「CLI 链路利用率」，查看进度和结果后再导出。
  • 确认结果后导出 PNG 或会话快照。
