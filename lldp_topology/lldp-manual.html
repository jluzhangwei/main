LLDP 拓扑工具使用手册 v2

概览

一句话：支持 CSV / SQL / CLI 三种导入，自动去重生成拓扑，支持分层布局、过滤定位、多选操作、链路利用率刷新、导出 PNG 与会话快照。

近期更新（关键行为）

  • 批量右键追加 多选设备后右键「追加查询」已支持批量顺序执行，固定按单轮补链路。
  • 链路利用率过滤 新增「利用率过滤」，输入阈值后只显示高于阈值的链路与相关节点，点「恢复显示」即可取消。
  • 链路缓存管理 新增「删除链路缓存」，会同时清空后端缓存和页面上的利用率颜色/标签。
  • 分区 + 分层 分区采用先命中优先（命中后不再参与后续分区）；每个分区内独立执行分层布局。
  • 状态快照 支持保存/导入服务器状态，服务器保留最近 15 个状态。

启动方式

  • 仅离线导入 CSV 直接打开 lldp.html。
  • 启用 SQL/CLI 需启动服务后访问 http://127.0.0.1:18080/lldp.html。
  • 依赖安装 pip install fastapi uvicorn pymysql。
  • 数据库配置 复制 .env.mysql.example 为 .env.mysql 并填写参数。

一键启动（推荐）

  • 脚本 /Users/zhangwei/python/lldp_topology/start_lldp_service.sh
  • 启动 cd /Users/zhangwei/python/lldp_topology && ./start_lldp_service.sh start
  • 状态 ./start_lldp_service.sh status
  • 日志 ./start_lldp_service.sh logs
  • 重启 ./start_lldp_service.sh restart
  • 停止 ./start_lldp_service.sh stop

SQL 导入（start_hostname）

  • 入口 导入方式选 SQL 导入，可点「导入输入」或「追加输入」。
  • 输入 设备名 start_hostname。
  • 执行方式 前端提交异步任务并轮询状态，任务完成后按下载地址读取结果，避免大结果直接回传导致超时。
  • 行为 「导入输入」会覆盖当前结果；「追加输入」会把本次结果合并到当前图中补链路。
  • 下载 支持下载本次查询的临时 CSV。
  • 临时文件 写入 tmp_csv/。

CLI 导入（SMC 跳板）

  • 入口 导入方式选 CLI 导入，可点「导入输入」或「追加输入」。
  • 参数 设备地址、递归层数、并发数、SSH 账号、SMC 参数、超时参数。
  • 本地记忆 自动记住上次输入（密码除外）。
  • 会话密码复用 页面不刷新时，CLI 可复用本次会话里最近一次输入的密码。
  • 流程 跳板登录 → 设备登录 → 厂商识别 → LLDP 采集 → 递归。
  • 过滤递归 支持「仅递归 172.* 网段」，跳过 10.* / 143.*。
  • 任务模式 CLI 查询采用异步任务执行，支持查看设备进度、总耗时，并可点击「取消查询」。
  • 追加 右键追加查询在 CLI 模式下固定只跑 1 轮，批量右键时按选中设备顺序执行。
  • 导出 支持导出本次 CLI 临时 CSV。

数据导入

  • CSV 选择文件、拖拽、或粘贴内容。
  • 去重 按 localhostname/localinterface/remotehostname/remoteinterface 去重。
  • 追加 追加 CSV 时保留当前坐标。
  • 去后缀 可去掉 hostname 域名后缀。

布局与分层

  • 布局 上下分层 / 强分散 / 圆形。
  • 分层关键字 逗号分层，斜杠同层，& 同层内全命中。示例：EDGE&CORE,CORE,AGGR/SPINE,LEAF/tor
  • 分区关键字 逗号分区，斜杠同区，& 同区内全命中。分区先匹配先归属，后续分区不重复匹配。
  • 选中重排 点击「重排选中设备」只对选中节点做局部布局，不影响其他节点和数据。
  • 默认 布局模式默认上下分层。

显示与交互

  • 标签 节点名全名/缩略，端口标签显示/隐藏。
  • 过滤 「不显示」关键词回车生效。
  • 定位 「定位设备」关键词回车多选。
  • 全选可见 一键选中当前可见节点，便于组合删除/重排。
  • 右键 固定/删除/复制名称/复制地址/链路利用率/追加查询。
  • 右键追加查询 按当前导入方式执行一次追加查询；SQL 优先用设备 IP，CLI 优先用管理地址。
  • 批量追加 多选设备后右键追加，会按选中设备顺序逐台执行并统一合并结果。
  • 单轮追加 右键追加查询固定只跑 1 轮，避免长时间递归。
  • 恢复 支持恢复删除节点，也可通过「恢复显示」取消临时视图过滤。

链路显示

  • 多链路合并 同一对节点可合并显示并统计条数。
  • CLI 链路利用率 采用任务化执行（提交任务 + 轮询进度），支持取消查询。
  • Zabbix 链路利用率 通过单独的 Zabbix 配置项接入，支持当前值与时间段最大值。
  • 右键利用率 可对单条链路、已选链路、已选设备相关链路右键执行链路利用率查询。
  • 默认优先级 右键链路利用率默认先查 Zabbix；若未配置或未命中，再回退到已配置的 CLI。
  • 缓存策略 已成功链路结果写入缓存，仅新增链路增量查询；CLI 和 Zabbix 缓存分开存储。
  • 缓存管理 支持下载链路缓存 CSV、删除链路缓存、下载利用率调试日志。
  • 利用率过滤 支持按阈值只显示高利用率链路，默认阈值 40%。

会话与导出

  • 图片 导出 PNG。
  • 会话 导出会话 / 导入会话（包含当前数据与坐标，可继续后续追加查询与操作）。
  • 服务器状态 支持保存状态 / 导入状态，服务器保留最近 15 个状态，适合阶段性回退与续做。

技能拆解（实现视角）

  • 登录设备 Skill SMC 跳板 + SSH 双阶段登录，自动处理 yes/no、old token、密码输入、分页关闭、超时与清理。
  • 获取数据 Skill SQL/CLI/CSV 三源统一成拓扑 CSV 结构，CLI 支持厂商识别与递归并发采集。
  • 拓扑生成 Skill 去重建图 + 分层/分区布局 + 交互过滤 + 增量查询 + 会话持久化。
  • 详版文档 见 LLDP_SKILLS.md。

常用操作流

  • 启动服务 → 打开页面 → 选择 SQL/CLI/CSV 导入。
  • 先用「导入输入」建立基线拓扑，再用「追加输入」补充新链路。
  • 需要针对一个或多个设备补链路时，先多选设备，再右键点「追加查询」（固定单轮）。
  • 生成拓扑后，用分层关键字和不显示规则调整视图。
  • 需要链路占用率时，可执行 CLI / Zabbix 链路利用率，或直接右键链路查询。
  • 需要聚焦高负载链路时，执行「利用率过滤」并输入阈值。
  • 阶段性完成后，导出 PNG / 会话，或先保存服务器状态再继续编辑。
