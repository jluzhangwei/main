# LLDP 拓扑项目技能总结

## 1) 登录设备 Skill（SMC Jump + SSH）

目标：稳定进入目标网络设备 CLI，为 LLDP 与端口利用率采集提供可复用会话。

关键能力：
- 双阶段登录：先连 SMC 跳板，再 `ssh user@device` 进入目标设备。
- 交互处理：自动识别并处理 `yes/no`、`old token (y/n)`、`password` 等提示。
- 提示符识别：基于设备提示符结束回显读取，防止截断或读挂。
- 分页关闭：按厂商自动下发禁分页命令（Huawei/NX-OS/Arista/IOS-XR）。
- 超时与清理：登录超时、命令超时、子进程回收、调试日志落地。

实现位置：
- `lldp_sql_service.py`：`SmcShellClient`、`detect_vendor`、`disable_paging`、命令执行封装。

常见风险与处理：
- 风险：`---- More ----` 导致输出不全。
- 处理：登录后立即执行禁分页；读取器识别分页并自动发送空格继续。

---

## 2) 获取数据 Skill（SQL / CLI / CSV 统一建模）

目标：把多来源数据统一为可建图的标准行结构，并保证可递归、可去重、可追加。

标准行结构（核心）：
- `递归次数`
- `源设备名称`
- `源设备端口`
- `目的设备名称`
- `目的设备端口`
- 扩展：`目的设备Vendor`、`目的设备IP`

关键能力：
- SQL 导入：异步任务化查询（创建任务 + 轮询状态 + 下载临时 CSV）。
- CLI 导入：
  - 自动识别厂商（`show version/dis version`）。
  - 厂商命令选择（Huawei / Cisco / Arista / IOS-XR）。
  - 多厂商 LLDP 解析器（含 system name / mgmt IP / local port / remote port 提取）。
  - 递归采集（层数 + 并发 + 去重 + 可选仅 172.* 递归）。
- CSV 导入：本地文件解析后与同结构数据合并。
- 去重策略：按 `localhostname/localinterface/remotehostname/remoteinterface` 去重。

实现位置：
- `lldp_sql_service.py`：SQL/CLI API、递归采集、厂商解析、临时 CSV 输出。
- `extract_lldp_neighbor_addresses.py`：独立 LLDP 文本解析工具。

常见风险与处理：
- 风险：设备名缺失，被 IP 顶替。
- 处理：优先 `System Name`，无名称时回退管理 IP，并在后续递归中回填。

---

## 3) 拓扑生成 Skill（布局、交互、增量能力）

目标：把标准行数据转换为可操作的拓扑视图，并支持持续补链路与运维分析。

关键能力：
- 建图：节点/边生成、链路去重、多链路合并。
- 布局：强分散、上下分层、圆形布局；支持分层关键字和分区关键字。
- 分区逻辑：先命中先归属；分区内独立分层；分区矩形区域与间距校验。
- 交互：过滤、仅显示、删除/恢复、右键追加查询、会话导入导出。
- 局部重排：`重排选中设备` 仅重排选中节点，不改写当前工作集。
- 链路利用率：CLI 任务化查询 + 增量缓存 + 标签/颜色映射。

实现位置：
- `lldp.html`：前端建图、交互、布局、会话、利用率渲染。
- `lldp_sql_service.py`：后端链路利用率任务与缓存接口。

常见风险与处理：
- 风险：大图拖拽卡顿。
- 处理：重边/拖拽降级渲染（如隐藏文字、简化边样式）、只对选区重排。

---

## 实战建议（按顺序）

1. 先用 SQL 或 CLI 建基线图（导入输入）。
2. 对关键节点执行右键追加查询（单轮）补链路。
3. 用分层/分区关键字做结构化排布。
4. 用 CLI 链路利用率做增量刷新。
5. 最后导出会话（可回放）和 PNG（可汇报）。
