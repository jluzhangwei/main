# Index.html 使用说明

## 1. 页面定位

`index.html` 是 LLDP 拓扑工具的轻量编辑页。

它的设计目标不是做实时采集，而是面向以下场景：

- 已经有现成拓扑数据，需要快速打开、整理、标注、导出
- 从 `lldp.html` 或其他来源导出的 `Json / CSV` 拓扑，需要做二次编辑
- 需要给他人一个更轻量、干净、少后端依赖的拓扑编辑入口
- 需要做概念图、汇报图、拓扑整理图，而不是现场采集

可以把它理解为：

- `lldp.html` 偏“采集 + 分析 + 运维”
- `index.html` 偏“编辑 + 整理 + 展示 + 导出”

## 2. 与 lldp.html 的核心区别

`index.html` 基于 `lldp.html` 裁剪而来，但有明确收口。

### 2.1 保留的能力

- CSV 导入 / 追加
- Json 导入 / 追加
- 自动建图
- 节点与链路编辑
- 节点组与组视图
- 路径查询
- 颜色、标签、过滤、分层、分区、合并链路等前端拓扑整理能力
- 导出 PNG / draw.io / Mermaid / Json / 链路汇总
- 保存状态 / 导入状态 / 撤销 / 恢复

### 2.2 刻意移除或隐藏的能力

`index.html` 不作为在线采集入口，因此默认不提供或不显示以下能力：

- SQL 导入
- CLI 导入
- NDMP 导入
- Zabbix 配置
- CLI 链路利用率实时查询
- Zabbix 链路利用率实时查询
- 右键当前利用率 / 区间利用率
- 右键追加查询

这意味着：

- `index.html` 更适合处理“已有拓扑数据”
- 如果你需要从设备或外部系统实时采集，请使用 `lldp.html`

## 3. 打开方式

直接双击本地文件。


## 4. 页面结构

`index.html` 页面主要分 4 个区域：

1. 顶部工具条
2. 左侧工作区
3. Cytoscape 拓扑画布
4. 各类弹窗与右键菜单

左侧工作区分为四大块：

- 导入数据
- 布局与分散
- 导出
- 链路利用率

其中“链路利用率”在 `index.html` 中主要用于**显示控制**，而不是在线采集。

## 5. 导入数据

## 5.1 导入方式

`index.html` 的导入方式收口为：

- `Json 导入`

其中：

- `导入`：覆盖当前工作集
- `追加`：把新内容合并进当前工作集

## 5.2 CSV 导入（移除入口）

CSV 适用于导入 LLDP 邻接关系表。

支持两种入口：

- 选择 CSV 文件
- 直接粘贴 CSV 内容

典型字段包括：

- `localhostname`
- `localinterface`
- `remotehostname`
- `remoteinterface`
- `sourceip`
- `remoteip`
- `depth`

如果字段名不是完全一致，页面会尝试自动识别，并允许你在左侧下拉框中重新指定：

- 本端设备列
- 对端设备列
- 本端端口列
- 对端端口列

## 5.3 Json 导入

Json 用于恢复一个完整的前端会话状态。

它通常包含：

- 原始拓扑行数据
- 节点坐标
- 当前过滤条件
- 分层 / 分区设置
- 节点颜色
- 节点组信息
- 链路描述
- 合并链路状态
- 撤销恢复上下文的一部分前端状态

这意味着 `Json 导入` 不是单纯导入链路，而是尽量恢复“上一次编辑现场”。

## 5.4 追加导入

追加模式用于以下场景：

- 在现有拓扑上补充一批新链路
- 把多个 CSV 或 Json 结果拼成一张更大的图
- 保留当前大部分布局，仅把新节点拖进来继续整理

当前追加逻辑会尽量做到：

- 旧节点保留原位置
- 新节点按导入内容附加
- 新增节点默认选中，方便继续拖动整理
- 已删除设备不会因为重复导入自动强行回到画面中

## 6. 拓扑生成与布局

这一组按钮负责从导入数据生成可视拓扑，并对布局做结构整理。

## 6.1 生成拓扑

`生成拓扑` 会将当前工作集重新转换成节点和链路。

建图时会处理：

- 设备名与 IP 归一
- 节点去重
- 多条链路合并
- 已有坐标继承
- 节点组与手工节点恢复
- 链路描述恢复

## 6.2 适配画布 / 居中放大

- `适配画布（慎用）`：把当前可见拓扑整体 fit 到画布
- `居中 + 放大`：将视图重新聚焦到图中心，便于重新观察

“慎用”的原因是图很大时，fit 可能改变你当前观察节奏。

## 6.3 合并多条链路

此按钮控制两个节点之间的多条物理链路是否汇聚显示。

- 开启：两台设备之间的多条链路可以合并成一条逻辑链路
- 关闭：每条物理链路都单独显示

开启后更适合看大图；关闭后更适合做精细链路检查。

## 6.4 重排选中设备

只对当前选中的节点做局部重排，不强制影响整图。

适用于：

- 某一块拖乱了，想局部整理
- 某个节点组附近需要重新排一下
- 不希望整图重新布局

## 6.5 查找路径

`查找路径` 基于当前可见拓扑进行无向图路径搜索。

支持：

- 源节点
- 目的节点
- 必经点
- 备份路径数量

结果会：

- 在图上高亮路径
- 在弹窗里列出路径节点与端口对
- 支持“只保留路径节点”

这适合做故障路径分析、业务路径确认、汇报时的链路抽取。

## 6.6 对齐与等间距

页面提供一组纯前端整理按钮：

- 水平对齐
- 垂直对齐
- 左对齐 / 右对齐
- 上对齐 / 下对齐
- 水平等间距
- 垂直等间距

这些功能对“概念图整理”和“汇报图美化”非常有用。

它们不会修改原始链路数据，只会调整当前节点坐标。

## 7. 视图过滤与定位

## 7.1 定位设备

`定位设备` 用于从当前图中快速找到设备。

表达式支持：

- 逗号：分层匹配
- 斜杠：同组 OR
- `&`：多个关键词同时命中

这个输入框偏“查找并选中”。

## 7.2 不显示

`不显示` 会隐藏命中的设备。

逻辑已统一为：

- 仅按**设备名称**匹配
- 不按 IP 地址参与匹配

这样可以避免像 `10-` 这种关键词误伤带 `10.x.x.x` 地址的设备。

## 7.3 只显示

`只显示` 与“不显示”相反，只保留命中的设备，其余隐藏。

同样支持：

- 逗号分层
- 斜杠同组
- `&` 全命中

## 7.4 只显示172设备

这个按钮用于快速收口到管理地址在 `172.*` 网段的普通设备。

当前逻辑：

- 只作用于普通节点
- 节点组不参与 `172.*` 判断
- 节点组不会因为这个按钮被一起隐藏

## 7.5 单链路节点

该功能会自动找出当前可见拓扑中“只与 1 台邻居设备相连”的节点，并高亮这些节点。

用途：

- 找悬挂节点
- 找边缘接入设备
- 快速识别单归属设备

## 7.6 返回主视图

如果当前在节点组视图中，可以通过该按钮返回主视图。

## 8. 删除、恢复与回退

## 8.1 删除类能力

支持：

- 删除选中节点
- 仅保留选中节点
- 仅保留邻接节点

删除是前端视图级删除，不会改原始 CSV 数据源。

## 8.2 恢复删除设备

删除的设备会进入删除栈，后续可以恢复。

这使得你可以大胆整理，不必担心一次误删后只能重导。

## 8.3 撤销 / 恢复

页面支持有限步数的编辑回退。

典型可回退的动作包括：

- 删除
- 移动节点
- 对齐 / 分散
- 手工加节点 / 加链路
- 颜色修改

它用于弥补 Cytoscape 画布交互中“误拖动、误操作”的问题。

## 8.4 保存状态 / 导入状态

这是服务器侧状态快照能力。

和 Json 的区别是：

- `Json 导出 / 导入`：偏文件交换
- `保存状态 / 导入状态`：偏同一服务上的阶段性保存

适合：

- 保存整理进度
- 不同阶段做快照
- 大图整理到一半，先落一个版本

## 9. 导出能力

## 9.1 导出 PNG

用于直接导出当前画面，用于汇报、截图、发群里。

## 9.2 导出链路汇总

按当前显示状态导出链路汇总 CSV。

如果链路已合并，导出为设备级汇总；
如果链路未合并，导出为端口级明细。

## 9.3 导出 draw.io

导出为 draw.io 可导入文件，适合继续到绘图工具做二次排版。

## 9.4 导出 Mermaid

导出当前拓扑为 Mermaid 文本，便于：

- 文档嵌入
- Markdown 展示
- 在线 Mermaid 渲染

## 9.5 导出 Json

导出当前完整编辑状态，供后续：

- 再次导入
- 团队协作
- 归档版本

## 10. 清理能力

清理区不是删除图上节点，而是清理运行期数据。

包括：

- 删除临时数据
- 删除连接配置
- 删除链路缓存

在 `index.html` 场景下，这些能力更多是“清理历史残留”，不是日常高频动作。

## 11. 链路利用率区在 index.html 中的角色

`index.html` 中仍保留了部分链路利用率显示控件，但它的职责和 `lldp.html` 不同。

### 11.1 在 index.html 中主要保留的是显示控制

包括：

- 链路拥塞分析
- 负载分担分析
- 显示利用率标签
- 显示 TX/RX
- 显示总带宽
- 显示链路描述

它们的用途是：

- 控制已经导入或恢复出来的数据如何显示
- 对图进行更适合展示的视觉切换

### 11.2 不建议把 index.html 当实时利用率采集页

虽然底层部分逻辑仍存在，但页面在轻量模式下已经隐藏了：

- 在线利用率查询入口
- Zabbix 配置入口
- 过滤与调试下载入口

因此从产品定位上，`index.html` 应视为：

- “利用率展示页”
- 而不是“利用率采集页”

## 12. 右键菜单能力

`index.html` 的右键菜单保留了大量前端编辑能力。

### 12.1 编辑类

- 固定位置
- 删除设备
- 删除链路
- 展开链路
- 添加描述

### 12.2 颜色类

- 自定义颜色
- 最近使用颜色
- 常用颜色
- 更多颜色
- 清除颜色

### 12.3 结构类

- 设为必经点
- 增加链路
- 设备合组
- 合组展开 / 收起
- 移出节点组
- 返回主视图
- 修改组名

### 12.4 复制类

- 复制名称
- 复制地址
- 复制链路

### 12.5 手工建模类

- 添加普通节点
- 添加 Cloud 节点

这意味着 `index.html` 可以脱离实时采集，单独用于：

- 概念图补充
- 汇报图加工
- 局部修图

## 13. 手工节点与手工链路

## 13.1 添加节点

支持新增两类节点：

- 普通节点
- Cloud 节点

Cloud 节点用于表达：

- 外部网络
- Internet
- 业务云
- 抽象区域

它不是 LLDP 自动发现出来的设备，而是人工建模元素。

## 13.2 增加链路

选择两台设备后，可以右键手工增加链路。

输入：

- 源端口
- 目的端口

保存后会直接追加到当前工作集，并在图上立即显示。

## 13.3 链路描述

对任意链路可追加描述。

描述会与链路一起保存、导出、随 Json 会话恢复。

适合补充：

- 业务说明
- 专线编号
- 备注信息
- 人工标识

## 14. 节点组与组视图

`index.html` 保留节点组合组能力。

支持：

- 多选节点后合组
- 手工组重命名
- 合组展开 / 收起
- 进入组视图
- 从节点组移出设备
- 返回主视图

节点组适合做：

- 业务域抽象
- 站点抽象
- 汇报图简化
- 设备簇折叠

## 15. 路径查询

路径查询是 `index.html` 中一个非常适合展示和分析的能力。

支持：

- 自动填充已选中的源 / 目的节点
- 设定多个必经点
- 设定备份路径数量
- 图上高亮
- 弹窗路径清单
- 只保留路径节点

对于需要导出给 draw.io 或汇报 PPT 的场景，它可以快速把大图裁剪成一条重点业务路径。

## 16. 适合 index.html 的典型工作流

推荐工作流如下：

1. 从 `lldp.html`、外部 CSV 或历史 Json 获取拓扑数据。
2. 在 `index.html` 中导入 Json 或 CSV。
3. 生成拓扑并做局部重排。
4. 使用过滤、只显示、路径查询把重点范围收出来。
5. 使用节点组、颜色、Cloud 节点、手工链路、链路描述补充说明。
6. 使用对齐和等间距整理成适合展示的版式。
7. 导出 PNG / draw.io / Mermaid / Json。

## 17. 使用建议

### 17.1 什么时候用 index.html

适合：

- 已有数据、需要快速出图
- 需要整理视觉效果
- 需要做汇报版本
- 需要概念图补充
- 不想暴露采集配置给最终使用者

### 17.2 什么时候不要用 index.html

不适合：

- 现场实时采集 LLDP
- 查询 Zabbix 当前告警或实时利用率
- 做 SQL / CLI / NDMP 在线导入
- 做设备级在线诊断

这些应回到 `lldp.html`。

## 18. 总结

`index.html` 的本质不是“缩水版首页”，而是**面向拓扑整理与展示的轻量工作台**。

它保留了拓扑编辑最有价值的前端能力：

- 导入已有数据
- 自动建图
- 手工补图
- 结构整理
- 路径抽取
- 节点组合并
- 多格式导出

同时主动收口在线采集、配置、调试类能力，避免把编辑页做成一个过于复杂的运维主控台。

如果你的目标是：

- 采集数据，用 `lldp.html`
- 整理数据、展示拓扑、导出成果，用 `index.html`

这就是两个页面最合理的分工。
