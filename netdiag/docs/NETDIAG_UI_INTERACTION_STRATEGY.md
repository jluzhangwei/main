# NetDiag UI 实时交互方略

## 目标
用最少控件保持诊断连续且可观测：
- 聊天输入作为唯一主入口
- 所有人工确认动作都在 AI 对话卡片中点击
- 右侧固定实时观测区持续展示“下一步 + 进度 + 回显”

## 按顺序执行策略
1. 对话解析：
- 从聊天中提取 `question`、`device_ip`、`fault_start`、`fault_end`
- 自动回填工作流字段

2. 守卫校验：
- 校验 `/netdiag/control` 的连接配置
- 若缺失则阻塞执行，并明确提示缺失项

3. 会话决策：
- 无会话时自动创建
- 检测到设备或时间窗漂移时，在对话中给“新会话/继续当前会话”二选一

4. Baseline 一次化：
- 仅在未采集时执行
- 同会话默认不重复采集

5. 轮次循环：
- `plan -> approve -> execute -> analyze`
- 每步通过对话卡片按钮推进，且可由自动批准策略接管

6. 收敛结束：
- 读取 `stop_decision`
- 满足条件则结束会话，否则进入下一轮

## UI 简化规则
1. 单一输入路径：聊天框 + Enter 发送。
2. 单一主动作面：AI 对话动作卡。
3. 单一状态解释区：右侧“实时方略”提示。
4. 步骤状态固定三态：`待执行 / 进行中 / 完成`。
5. 策略配置用弹出菜单：`自动批准`、`经验库`。
6. 聊天记录按会话持久化：刷新保留，只有确认新建会话时清空当前对话区。

## 右侧实时观测契约
右侧固定展示 4 块：
1. 顶部步骤条（全局阶段）
2. 实时方略卡（6 步顺序 + 下一步 + 提示）
3. 对话识别上下文（设备与故障时间窗）
4. 设备执行回显尾部

## 验收标准
1. 新用户只用聊天输入与回车即可启动诊断。
2. 任何需要人工确认的步骤都在 AI 对话内给出可点击按钮。
3. 用户始终能从右侧方略卡看懂“现在在哪一步、下一步是什么”。
4. 页面不存在与对话动作卡重复的主控按钮区。
