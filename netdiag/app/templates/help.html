NetDiag Help

Extract buffer logs from network devices by time range and run optional AI diagnosis.

Core Features

  • Single or batch device log extraction (text input)
  • Connection modes: Direct SSH (asyncssh) and SMC jump mode (pty + subprocess)
  • Auto vendor/profile handling for Cisco/Arista/Huawei/Palo Alto
  • Server-local time window input with device clock offset conversion
  • Hit lines with context preservation and merged overlapping ranges
  • Download outputs: per-device log, summary.json, and zip package
  • Debug mode shows login/command/response transcript
  • AI analysis with multi-provider models and markdown output

Quick Start

  1. Open Create Task, choose direct or smc.
  2. Fill device credentials and time range (YYYY-MM-DD HH:MM:SS).
  3. Paste device list in the address box (one per line).
  4. Create task and monitor progress in task detail.
  5. Download filtered.log, summary.json, and download.zip.
  6. Optionally run AI analysis on task detail page.

SMC Mode Notes

  • Enter jump shell first, then SSH to target device in the same PTY session.
  • Auto-handles yes/no and password prompts.
  • Recognizes common device prompts before command execution.
  • One isolated SMC sub-session per device; cleaned up at task end.

Output Layout

output/{task_id}/
  summary.json
  download.zip
  dev-1/
    raw.log
    filtered.log
    meta.json
    debug.log (debug mode)

Troubleshooting

  • CERTIFICATE_VERIFY_FAILED: start service with ./run.sh to load CA bundle defaults.
  • No log hits: verify device clock, timezone, and input window.
  • Unsupported command: fallback commands are attempted; failure reason appears in task detail.

NetDiag Help

用于从网络设备 buffer log 中按时间范围提取日志，并支持 AI 诊断分析。

核心功能

  • 单台与批量设备日志提取（文本输入）
  • 连接方式：Direct SSH（asyncssh）和 SMC 跳板模式（pty + subprocess）
  • 自动识别设备厂商/系统（Cisco/Arista/Huawei/Palo Alto）并执行对应命令
  • 按“服务器本机时间”输入时间窗，自动计算设备时钟偏移（offset）后过滤
  • 命中日志保留前后上下文行，重叠/相邻区间自动合并去重
  • 任务结果下载：单设备日志、任务 summary、整包 zip
  • Debug 模式可查看登录/命令执行/设备返回
  • AI 分析支持多模型源、提示词模板、Markdown 结果展示

使用流程

  1. 进入 Create Task 页面，选择 direct 或 smc。
  2. 填写设备账号和时间范围（格式：YYYY-MM-DD HH:MM:SS）。
  3. 在设备地址文本框粘贴设备列表（每行一台）。
  4. 点击创建任务后进入任务详情页，页面会自动轮询进度。
  5. 任务完成后下载 filtered.log、summary.json、download.zip。
  6. 可在任务详情页执行 AI 分析，查看渲染后的 Markdown 诊断结果。

SMC 模式说明

  • 先进入跳板 shell（自动处理 yes/no 和 token 提示）
  • 再从跳板 ssh 到目标设备（自动发送密码，最多重试）
  • 识别到设备提示符后执行版本/时间/日志命令
  • 每台设备独立 SMC 子会话，并发执行，任务结束自动清理进程

输出目录结构

output/{task_id}/
  summary.json
  download.zip
  dev-1/
    raw.log
    filtered.log
    meta.json
    debug.log (debug 模式)

常见问题

  • 证书错误 CERTIFICATE_VERIFY_FAILED：优先使用 ./run.sh 启动（会自动注入 CA bundle）。
  • 日志未命中：检查设备时钟、时区、输入时间窗，必要时在 Debug 中查看 show/display clock 返回。
  • 命令不兼容：系统会先做厂商识别，再回退到可用命令；失败原因会记录在任务详情 reason。
