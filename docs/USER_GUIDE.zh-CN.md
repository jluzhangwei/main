# NetOps AI V1 用户说明书（图文）

本文面向一线运维与值班工程师，帮助你快速上手 NetOps AI V1 的日常诊断、查询和配置协作。

## 1. 页面总览

![工作台总览](./images/user-manual/01-workbench.png)

- 左侧图标栏：页面导航。
- 中间区域：AI 诊断对话、执行卡片、用户输入。
- 右侧区域：命令回显、结果细节。

## 2. 首次使用（推荐顺序）

1. 进入“连接控制”，填写设备地址、用户名、密码。
2. 选择任务模式：诊断排障 / 状态查询 / 配置变更。
3. 选择自动化等级：极低风险 / 中风险可执行 / 高风险可执行。
4. 返回“诊断工作台”，输入需求并发送。
5. 在执行卡片与右侧回显区查看全过程证据。

## 3. 关键页面说明

### 3.1 连接控制

![连接控制](./images/user-manual/02-control.png)

用途：创建会话、设置设备连接参数、选择任务模式与自动化等级。

要点：
- 默认建议先使用“诊断排障 + 中风险可执行”。
- 配置变更前确认账号权限与回滚方案。

### 3.2 命令执行控制

![命令执行控制](./images/user-manual/03-command-policy.png)

用途：维护命令阻断、放行与风险分级规则。

要点：
- 阻断规则优先级最高。
- 未命中放行且命中高风险时，会进入确认流程。

### 3.3 会话历史

![会话历史](./images/user-manual/04-sessions.png)

用途：回看与恢复历史会话。

交互：
- 单击：选择会话并查看右侧快照。
- 双击：进入并恢复该会话继续对话。

### 3.4 流程追踪

![流程追踪](./images/user-manual/05-service-trace.png)

用途：按步骤查看完整执行链路和耗时。

能力：
- 支持前序/后续跳转。
- 支持 Play 回放，按步骤时间自动演示全过程。
- 点击步骤后，步骤明细会自动滚动到对应行。

### 3.5 学习规则（版本级）

![学习规则](./images/user-manual/06-learning.png)

用途：维护失败阻断 / 替代改写规则，减少重复试错。

要点：
- 规则作用域为版本指纹。
- 支持手动编辑与自动学习来源区分。

### 3.6 AI 设置

![AI 设置](./images/user-manual/07-ai-settings.png)

用途：配置 API Key、模型、故障切换策略与提示词展示。

要点：
- API Key 默认保存到服务器。
- 可手动删除 Key；删除后模型不可用。

## 4. 标准操作流程（值班推荐）

1. 创建会话并验证设备可达。
2. 在工作台输入目标（例如：检查端口/检查协议/执行修复）。
3. 观察 AI 卡片与右侧回显，确认证据是否充分。
4. 涉及变更时进行确认或使用自动化策略。
5. 在流程追踪页复盘耗时和关键判定点。
6. 在会话历史页沉淀可复用案例。

## 5. 常见问题

- Q: 为什么有命令需要确认？
  - A: 命令命中高风险策略或未命中放行策略时，会进入确认流程。

- Q: 为什么刷新后看不到刚才会话？
  - A: 请到“会话历史”刷新并恢复；系统已支持历史会话快照与恢复。

- Q: 为什么 AI 无法给出结论？
  - A: 请先检查 AI 设置中的 API Key、模型联机状态与错误信息。
