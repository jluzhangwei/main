# V4 预研计划（SSO / 多租户 / 审批体系）

## 目标

在 V3 多设备编排能力基础上，推进企业级平台化：

- SSO/OIDC 登录
- 多租户隔离
- 审批人/角色组织体系
- 审计合规增强

## 1. 架构方向

### 1.1 身份认证

- 引入 OIDC（如 Keycloak / Azure AD / Okta）
- 后端验证 JWT（`iss/aud/exp/sub`）
- API Key 模式保留为“系统到系统调用”通道

### 1.2 多租户

- 数据模型增加 `tenant_id`
- 任务、设备、审批、审计均按 `tenant_id` 隔离
- API 层引入 `tenant scope` 校验

### 1.3 审批体系

- 审批流模板化（按风险等级、命令组类型、设备域）
- 支持多人会签、超时、升级审批
- 审批动作全量审计（审批前后 diff）

## 2. 关键设计要点

### 2.1 秘钥与凭据安全

- 设备凭据接入 KMS/Vault
- API Key 仅保存 hash，支持轮换与过期
- 敏感字段日志脱敏（密码、token、密钥片段）

### 2.2 可观测性

- Trace ID 跨任务链路透传
- 事件总线指标：任务延迟、审批等待时长、执行成功率
- RCA 评分可解释性输出（权重、证据、推理链）

### 2.3 可扩展性

- 任务执行器从单进程迁移到队列（Redis Stream / RabbitMQ / Kafka）
- 设备连接池与限流策略（按厂商/网络域）

## 3. 分阶段里程碑

1. `V4.1` OIDC + JWT 鉴权落地（保留 API Key）
2. `V4.2` 多租户数据隔离与权限域
3. `V4.3` 审批模板引擎 + 会签
4. `V4.4` 审计合规报表增强（租户维度）
5. `V4.5` 队列化执行与水平扩展

## 4. 风险与应对

- 风险：现有 API Key 客户端大量在用
  - 应对：双栈鉴权并行，分阶段迁移
- 风险：租户隔离改造范围大
  - 应对：先做读路径隔离，再切写路径
- 风险：审批流复杂导致交付慢
  - 应对：先交付通用模板 + 单级审批，再扩展会签

## 5. 验收指标建议

- 租户越权访问拦截率 100%
- 高风险命令审批漏拦截 0
- SSO 登录成功率 > 99.9%
- 任务调度稳定性在 50~200 设备规模无明显抖动
